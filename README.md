# Atlas

Atlas is an AI-powered immersive 3D history experience built for the Education track of our hackathon.

The goal is to make learning more interactive, accessible, and personalized by letting users generate and explore historical environments using natural language prompts.

Using 3D Gaussian Splatting, Atlas reconstructs visually rich, spatial scenes that feel realistic and immersive, helping users build intuition and context instead of just memorizing facts.

## Idea

Users can enter prompts like:

- “A marketplace along the Silk Road in the 14th century”
- “An ancient Roman street during the day”
- “A medieval European town square”

Atlas then generates an explorable 3D environment based on that prompt, allowing users to step into history and learn through exploration.

## Experience Goals

Atlas is designed to feel:

- immersive
- calm and intuitive
- curiosity-driven
- personalized
- visually grounded in reality

## Core Features

### 1. Prompt-to-Environment Generation
- Users enter a natural language prompt describing a place and time
- The system generates a matching 3D environment
- Scenes are designed to feel coherent, believable, and spatial

### 2. Immersive Exploration
- Users can move through the environment freely
- They can examine objects and spaces from different angles
- The experience feels like a place, not just a static image

### 3. Contextual Points of Interest
- Key objects or locations can be interacted with
- Each interaction provides short, simple context
- Explanations focus on daily life, environment, and usage

### 4. AI Chatbot / Narrator
- A built-in chatbot is always available during exploration
- Users can ask questions about what they are seeing in real time

Example questions:
- “What is this building used for?”
- “Who are the people here?”
- “What time period is this?”
- “Why are things arranged this way?”

The chatbot is designed to:
- respond clearly and conversationally
- stay grounded in the current environment
- provide helpful context without overwhelming the user
- adapt to what the user is looking at or interacting with

### 5. Guided Interpretation
- Optional prompts guide the user’s attention
- Example: “Notice how goods are arranged”
- Example: “Look at how dense the buildings are”
- This encourages learning through observation

## Handling Prompt Quality

Atlas is designed to handle unclear or imperfect prompts gracefully.

- If a prompt is too vague, the system asks a clarifying question or suggests a more specific version
- If a prompt is unrealistic or contradictory, the system gently reinterprets it and explains the adjustment
- If a prompt is incomplete, the system fills in reasonable defaults

The goal is to make the experience robust, user-friendly, and flexible.

## Example User Flow

1. User enters a prompt  
2. Atlas generates a 3D environment  
3. User is placed inside the scene  
4. User explores and interacts with points of interest  
5. User asks questions to the chatbot during exploration  
6. The chatbot provides real-time contextual explanations  
7. User leaves with a stronger mental model of the place

## Why Atlas Stands Out

Atlas combines:

- AI-driven generation of learning environments
- 3D Gaussian Splatting for immersive visuals
- a real-time interactive chatbot guide
- personalized, curiosity-driven learning
- a polished product-focused experience

Our goal is to make history feel real, explorable, and conversational.

For the hackathon demo, we are focusing on one highly polished example prompt and environment.
